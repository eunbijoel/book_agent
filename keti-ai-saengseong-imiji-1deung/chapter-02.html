제2장: 승리의 기술: 경량화, 최적화, 설명 가능성의 통합

세 개의 산을 동시에 오르기

AI 생성 이미지를 정확히 탐지하는 것만으로는 충분하지 않다. 스마트폰 속 카메라가 실시간으로 영상을 처리해야 하고, 자율주행차가 도로 위에서 신뢰할 수 없는 센서 신호를 즉각 거부해야 하며, 로봇이 배터리 몇 시간으로 연속 작동해야 하는 현실 속에서, 우리가 마주하는 도전은 단순한 정확도 문제가 아니다.

KETI 연구진이 CVPR IEEE 저전력 컴퓨터 비전 챌린지에서 1위를 차지한 이유는 세 가지 산을 동시에 오르는 데 성공했기 때문이다. 경량화(Lightweight), 최적화(Optimization), 설명 가능성(Explainability) — 이 세 가지 기술이 어떻게 통합되었는지 이해하는 것이 이 장의 핵심이다.

첫 번째 산: 모델 경량화의 기술

경량화란 무엇인가?

경량화는 단순히 모델을 "작게" 만드는 것이 아니다. 이는 제한된 자원 환경에서 최대한의 성능을 유지하면서 계산 복잡도를 줄이는 지능형 구조 설계이다.

기존의 고성능 AI 생성 이미지 탐지 모델들은 수십억 개의 파라미터를 가지고 있었다. CLIP(Contrastive Language-Image Pre-training)이나 BLIP(Bootstrapping Language-Image Pre-training) 같은 비전-언어 모델(VLM)은 수십억 파라미터로 구성되어 있으며, 이들을 엣지 디바이스에 탑재하려면 수십 GB의 메모리가 필요했다. 스마트폰은 보통 4-8GB의 RAM을 가지고 있고, 임베디드 시스템은 그보다 훨씬 적다.

KETI가 적용한 경량화 기법은 세 가지 축으로 구성된다.

1. 지식 증류(Knowledge Distillation)

지식 증류는 대형 모델(교사 모델)이 학습한 지식을 소형 모델(학생 모델)에게 전이하는 기법이다. 단순히 최종 예측값만 전달하는 것이 아니라, 모델이 어떻게 생각하는지 그 과정을 배우게 한다.

KETI 연구진은 다음과 같은 방식을 적용했다:

  • 교사 모델: 고성능 VLM(예: BLIP-2)으로 훈련
  • 학생 모델: 파라미터 수를 1/10 이하로 축소한 경량 모델
  • 지식 전이: 교사 모델의 중간 계층(intermediate layer) 활성화 값을 학생 모델이 모방하도록 훈련

실제 결과는 놀라웠다. 교사 모델이 95% 정확도를 달성했을 때, 파라미터 수를 90% 감소시킨 학생 모델도 91-93% 정확도를 유지했다. 이는 "경량화 = 성능 저하"라는 통념을 깨뜨리는 결과였다.

2. 양자화(Quantization)

양자화는 모델의 파라미터를 더 낮은 정밀도로 표현하는 기법이다. 일반적으로 신경망의 가중치는 32비트 부동소수점(float32)으로 저장된다. 양자화를 통해 이를 8비트 정수(int8)나 4비트로 축소할 수 있다.

이것이 얼마나 효과적인지 다음 표를 보면 명확하다:

정밀도 메모리 크기 추론 속도 정확도 손실
Float32 1.0x 1.0x —
Float16 0.5x 1.5–2.0x ~0.5%
Int8 0.25x 2.5–4.0x ~1–2%
Int4 0.125x 3–5x ~2–4%

KETI는 동적 양자화(dynamic quantization)와 정적 양자화(static quantization)를 혼합 적용했다. 모델의 초기 계층은 정보 손실이 크므로 float16으로 유지하고, 후기 계층은 int8로 양자화했다. 결과적으로 메모리 사용량은 75% 감소했지만, 정확도는 단 1%만 하락했다.

3. 프루닝(Pruning)

프루닝은 신경망에서 덜 중요한 연결을 제거하는 기법이다. 인간의 뇌도 발달 과정에서 불필요한 신경 연결을 제거하는데, 이와 유사한 원리다.

KETI 연구진은 구조적 프루닝(structured pruning)을 적용했다. 개별 가중치가 아니라 전체 뉴런이나 채널을 제거하는 방식으로, 하드웨어 가속에 더 유리하다.

예를 들어, 원본 모델이 64개의 채널을 가진 컨볼루션 계층을 가지고 있었다면, 프루닝 후에는 32개 또는 16개로 축소된다. 이렇게 하면 단순히 연산량이 줄어드는 것뿐 아니라, 메모리 접근 패턴도 개선되어 실제 하드웨어에서의 성능 향상이 더 크다.

세 기법의 조합으로, KETI는 원본 모델 대비 모델 크기 87% 감소, 추론 시간 73% 단축을 달성했다.

두 번째 산: 저전력 추론 최적화

에너지 효율성이 왜 중요한가?

경량화된 모델도 여전히 에너지를 소비한다. 엣지 디바이스의 배터리는 유한하다. 스마트폰에서 1초에 1W를 소비하는 모델이 있다면, 4000mAh 배터리는 단 1-2시간 만에 방전된다.

KETI의 저전력 최적화는 다음 세 가지 영역을 포함한다.

메모리 계층 최적화

데이터를 메모리에서 꺼내오는 것 자체가 연산보다 더 많은 에너지를 소비한다. 현대 프로세서에서 메인 메모리에 접근하는 데 드는 에너지는 간단한 부동소수점 연산의 100-1000배에 달한다.

KETI 연구진은 타일화(tiling) 기법을 적용했다. 모델의 계산을 작은 블록 단위로 나누어, CPU 캐시에 들어갈 수 있는 데이터만 처리하도록 재구성했다.

구체적으로, 이미지 크기 224×224 픽셀의 입력을 처리할 때:

  • 기존 방식: 전체 이미지를 메모리에 로드 → 계산 → 결과 저장
  • 타일화 방식: 56×56 픽셀 블록 16개로 나눔 → 각 블록을 캐시에 로드 → 계산 → 결과 누적

이 방식으로 메모리 대역폭 요구량을 40% 감소시켰고, 결과적으로 전력 소비는 32% 줄어들었다.

배치 정규화 폴딩(Batch Normalization Folding)

신경망의 배치 정규화(Batch Normalization) 계층은 훈련 중에는 필수적이지만, 추론 단계에서는 이전 계층의 가중치에 통합할 수 있다. 이를 "폴딩"이라 한다.

예를 들어:

y = BatchNorm(Conv(x))

이를 다음과 같이 변환할 수 있다:

y = Conv'(x)  // Conv의 가중치가 이미 정규화를 포함

이 단순한 변환으로 추론 시 배치 정규화 연산이 완전히 제거되어, 각 이미지당 약 5-8% 추론 시간 단축과 함께 메모리 접근 패턴이 개선된다.

연산 정밀도 적응(Adaptive Precision)

모든 계층이 동일한 정밀도로 연산할 필요는 없다. KETI는 계층별 적응형 정밀도를 도입했다:

  • 초기 계층(특징 추출): float16 — 입력 이미지의 다양한 특징을 포착하는 데 높은 정밀도 필요
  • 중간 계층(특징 결합): int8 — 특징 결합에는 낮은 정밀도로도 충분
  • 최종 계층(분류): int8 또는 int4 — 이진 분류(생성/실제)에는 매우 낮은 정밀도로도 가능

이 접근법으로 평균 전력 소비를 28% 감소시키면서도 정확도는 유지했다.

실제 디바이스에서의 성능

KETI 연구진은 다양한 엣지 디바이스에서 최적화를 검증했다:

디바이스 메모리 추론 시간 전력 소비
Snapdragon 8 Gen2 8GB 45ms 0.8W
Apple Neural Engine — 38ms 0.6W
Qualcomm Hexagon 1GB 120ms 0.4W
Raspberry Pi 4 4GB 850ms 0.2W

심지어 저사양 임베디드 시스템인 라즈베리파이 4에서도 850ms 내에 추론을 완료할 수 있었다. 이는 초당 1장 이상의 이미지를 처리 가능함을 의미한다.

세 번째 산: 설명 가능한 AI의 통합

XAI가 필수인 이유

정확도 95%의 모델이 어떤 이미지를 "생성됨"이라 판단했을 때, 우리는 왜 그렇게 결정했는지 알아야 한다. 특히 중요한 의사결정 상황에서는 더욱 그렇다.

자율주행차가 도로 위의 신호를 "합성 신호"라고 판단하거나, 로봇이 물체를 "가짜"라고 인식한다면, 그 근거를 제시할 수 있어야 한다. 이것이 설명 가능한 AI(XAI, Explainable AI)이다.

KETI의 XAI 구현 전략

일반적으로 XAI는 추가 오버헤드로 여겨진다. 하지만 KETI는 XAI를 모델 구조의 핵심 요소로 설계했다.

1. 계층적 어텐션 메커니즘(Hierarchical Attention Mechanism)

비전-언어 모델의 강점은 이미지의 특정 영역과 언어 설명을 연결할 수 있다는 것이다. KETI는 이를 경량화된 모델에도 적용했다:

입력 이미지 → 특징 추출 → 다중 스케일 특징 → 어텐션 가중치 계산
                                              ↓
                                        중요 영역 강조
                                              ↓
                                        분류 결정 + 설명 생성

어텐션 메커니즘은 모델이 이미지의 어느 부분을 보고 판단했는지 시각화할 수 있다. 예를 들어, AI 생성 이미지 탐지 모델이 어떤 이미지를 "생성됨"이라 판단했다면, 그 이유가 되는 영역(예: 부자연스러운 손가락, 이상한 배경 텍스처)이 하이라이트되어 표시된다.
